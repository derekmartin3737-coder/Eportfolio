Deployment

Publish for free with GitHub Pages

This site uses only static files, which makes it a strong fit for GitHub Pages. No build tools are required unless you decide to add them later.

01

Create a repository

Put this site in a GitHub repository. If you use a normal project repository, deploying from the root folder works well.

02

Push the site files

Commit index.html, styles.css, script.js, and the artifacts/ folder so the portfolio structure is visible.

03

Enable Pages in GitHub

In the repository settings, publish from your main branch and the root folder. GitHub will generate a public portfolio URL for you.
